Documentación del Código y Control de Versiones

Aprende a documentar correctamente tu código y cómo integrarlo con herramientas de versionamiento.

Introducción

La documentación del código es fundamental para que otros desarrolladores (o tú mismo en el futuro) puedan entender, mantener y ampliar un proyecto. Además, combinada con herramientas de control de versiones como Git, permite un seguimiento ordenado de cambios.

Tipos de Documentación

  • En línea: Comentarios cortos junto a líneas de código para aclarar su función.
  • De bloque: Explicaciones más amplias sobre secciones completas.
  • Externa: Manuales o guías que describen cómo funciona el software.

Buenas Prácticas

  • Escribir comentarios claros, breves y útiles.
  • Actualizar la documentación cuando cambie el código.
  • Evitar comentar lo obvio.
  • Usar un formato estándar (ej. JSDoc en JavaScript).

Documentación y Control de Versiones

Los sistemas como Git permiten incluir documentación junto con los cambios de código, asegurando que siempre esté actualizada. Ejemplos:

# Crear un commit con mensaje descriptivo
git commit -m "Agregada documentación a la función calcularTotal"

# Subir cambios documentados al repositorio remoto
git push origin main

Herramientas

Herramienta Uso
JSDoc Generar documentación automática para proyectos JavaScript.
Git Controlar cambios en el código y documentación.
Markdown Escribir documentación legible y bien formateada.
