Estándares de Codificación en JavaScript

Guía interactiva sobre buenas prácticas, nomenclatura y estilo para escribir código más claro y mantenible.

Introducción

Los estándares de codificación son un conjunto de reglas y recomendaciones que permiten escribir código consistente, legible y fácil de mantener. Seguir estos estándares mejora la colaboración entre desarrolladores y reduce errores.

Nomenclatura

Variables y Constantes

  • Usar camelCase para variables: let userName;
  • Usar UPPER_CASE para constantes: const MAX_USERS = 100;

Clases

  • Usar PascalCase para clases: class UserProfile {}

Métodos y Funciones

  • Usar camelCase: function calculateTotal() {}

Comentarios

  • Usar comentarios claros y concisos.
  • Comentarios de una línea: // Explicación breve
  • Comentarios de bloque: 
    /* 
  Explicación más detallada
  sobre el bloque de código
*/

Ejemplo de Código

// Constante en mayúsculas
const MAX_USERS = 100;

// Clase con PascalCase
class UserProfile {
  constructor(name) {
    this.name = name; // Variable en camelCase
  }

  // Método en camelCase
  displayInfo() {
    console.log(`Usuario: ${this.name}`);
  }
}

// Uso de la clase
let userName = "Ana";
let user = new UserProfile(userName);
user.displayInfo();
